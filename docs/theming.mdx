Docsfly comes with a powerful theming system that allows you to customize the appearance of your documentation site. You can use the built-in themes or create your own custom theme.

## Built-in Themes

Docsfly includes two built-in themes:

### Light Theme (Default)
- Clean, professional appearance
- High contrast for readability
- Optimized for daytime reading

### Dark Theme
- Easy on the eyes in low-light conditions
- Reduced eye strain
- Popular among developers

<Callout type="info" title="Theme Toggle">
Users can switch between light and dark themes using the theme toggle button in the header.
</Callout>

## Theme System

The theme system is built around a structured theme object that defines colors, typography, spacing, and other design tokens.

### Theme Structure

<CodeBlock language="typescript" filename="lib/theme.ts">
{`export interface DocsTheme {
  colors: {
    primary: string;
    secondary: string;
    accent: string;
    background: string;
    surface: string;
    text: {
      primary: string;
      secondary: string;
      muted: string;
    };
    border: string;
  };
  typography: {
    fontFamily: {
      sans: string;
      mono: string;
    };
    fontSize: {
      xs: string;
      sm: string;
      base: string;
      lg: string;
      xl: string;
      '2xl': string;
      '3xl': string;
      '4xl': string;
    };
  };
  spacing: {
    xs: string;
    sm: string;
    md: string;
    lg: string;
    xl: string;
    '2xl': string;
  };
  borderRadius: {
    sm: string;
    md: string;
    lg: string;
    xl: string;
  };
}`}
</CodeBlock>

## Using the Theme Provider

The `ThemeProvider` component manages the current theme and provides theme switching functionality.

<CodeBlock language="tsx" filename="app/layout.tsx">
{`import { ThemeProvider } from '@/lib/theme-provider';
import { ThemeToggle } from '@/components/theme-toggle';

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <body>
        <ThemeProvider>
          <div className="min-h-screen bg-white dark:bg-gray-900">
            <header>
              <ThemeToggle />
            </header>
            <main>{children}</main>
          </div>
        </ThemeProvider>
      </body>
    </html>
  );
}`}
</CodeBlock>

## Creating Custom Themes

You can create custom themes by extending the base theme structure:

<CodeBlock language="typescript" filename="lib/custom-theme.ts">
{`import { DocsTheme, defaultTheme } from '@/lib/theme';

export const customTheme: DocsTheme = {
  ...defaultTheme,
  colors: {
    ...defaultTheme.colors,
    primary: '#8b5cf6', // Purple
    accent: '#06b6d4',   // Cyan
    background: '#fafafa',
    surface: '#ffffff',
  },
  typography: {
    ...defaultTheme.typography,
    fontFamily: {
      sans: 'SF Pro Display, -apple-system, sans-serif',
      mono: 'SF Mono, Monaco, monospace',
    },
  },
};`}
</CodeBlock>

## Best Practices

<Callout type="info" title="Accessibility">
When creating custom themes, ensure sufficient color contrast ratios for accessibility. Use tools like WebAIM's contrast checker to verify your color choices.
</Callout>

### Color Contrast
- Maintain at least 4.5:1 contrast ratio for normal text
- Use at least 3:1 contrast ratio for large text
- Test both light and dark theme variants

### Typography
- Use system fonts for better performance
- Maintain consistent font scales
- Consider reading comfort for long-form content

### Responsive Design
- Test themes across different screen sizes
- Ensure touch targets are appropriate for mobile devices
- Consider different viewing distances

## Theme Persistence

The theme system automatically persists user preferences in localStorage:

<CodeBlock language="typescript">
{`// Theme preference is automatically saved
localStorage.setItem('docsfly-theme', 'dark');

// And restored on page load
const savedTheme = localStorage.getItem('docsfly-theme');`}
</CodeBlock>

<Callout type="success" title="Automatic Persistence">
Users' theme preferences are automatically saved and restored when they return to your documentation site.
</Callout>
