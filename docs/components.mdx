Docsfly comes with several built-in components to help you create rich, interactive documentation. All components are designed to work seamlessly with both light and dark themes.

## Callout

The `<Callout>` component is perfect for highlighting important information, warnings, or tips.

### Usage

<CodeBlock language="mdx">
{`<Callout type="info" title="Information">
This is an info callout with a title.
</Callout>

<Callout type="warning">
This is a warning callout without a title.
</Callout>`}
</CodeBlock>

### Examples

<Callout type="info" title="Information">
This is an info callout. Use it to provide helpful information to your readers.
</Callout>

<Callout type="warning" title="Warning">
This is a warning callout. Use it to alert readers about potential issues or important considerations.
</Callout>

<Callout type="error" title="Error">
This is an error callout. Use it to highlight critical errors or problems.
</Callout>

<Callout type="success" title="Success">
This is a success callout. Use it to confirm successful operations or positive outcomes.
</Callout>

### Props

- `type`: `"info" | "warning" | "error" | "success"` (default: `"info"`)
- `title`: Optional title for the callout
- `children`: Content of the callout

## CodeBlock

The `<CodeBlock>` component provides syntax highlighting and additional features for code examples.

### Usage

<CodeBlock language="mdx">
{`<CodeBlock language="javascript" filename="example.js">
{\`function hello() {
  console.log("Hello, world!");
}

hello();\`}
</CodeBlock>`}
</CodeBlock>

### Examples

<CodeBlock language="javascript" filename="example.js">
{`function hello() {
  console.log("Hello, world!");
}

hello();`}
</CodeBlock>

<CodeBlock language="bash">
{`# Install dependencies
bun install

# Start development server
bun run dev`}
</CodeBlock>

### Props

- `language`: Programming language for syntax highlighting
- `filename`: Optional filename to display above the code block
- `showLineNumbers`: Whether to show line numbers (default: `false`)
- `children`: Code content

## Tabs

The `<Tabs>` component allows you to organize content into multiple tabs.

### Usage

<CodeBlock language="mdx">
{`<Tabs items={[
  { 
    id: 'tab1', 
    label: 'Tab 1', 
    content: 'Content for the first tab' 
  },
  { 
    id: 'tab2', 
    label: 'Tab 2', 
    content: 'Content for the second tab' 
  }
]} defaultTab="tab1" />`}
</CodeBlock>

### Example

<Tabs items={[
  { 
    id: 'react', 
    label: 'React', 
    content: (
      <CodeBlock language="jsx">
        {`import React from 'react';

function MyComponent() {
  return <div>Hello from React!</div>;
}

export default MyComponent;`}
      </CodeBlock>
    )
  },
  { 
    id: 'vue', 
    label: 'Vue', 
    content: (
      <CodeBlock language="vue">
        {`<template>
  <div>Hello from Vue!</div>
</template>

<script>
export default {
  name: 'MyComponent'
}
</script>`}
      </CodeBlock>
    )
  },
  { 
    id: 'svelte', 
    label: 'Svelte', 
    content: (
      <CodeBlock language="svelte">
        {`<script>
  // Component logic here
</script>

<div>Hello from Svelte!</div>`}
      </CodeBlock>
    )
  }
]} />

### Props

- `items`: Array of tab objects with `id`, `label`, and `content`
- `defaultTab`: ID of the default active tab

## Customization

All components are built with Tailwind CSS and support both light and dark themes out of the box. You can customize their appearance by modifying the component files in the `/components` directory.

<Callout type="info" title="Theme Support">
All components automatically adapt to the current theme (light/dark) using Tailwind's dark mode classes.
</Callout>

## Creating Custom Components

You can create your own custom components and make them available in MDX files by adding them to the `mdx-components.tsx` file:

<CodeBlock language="tsx" filename="mdx-components.tsx">
{`import { MyCustomComponent } from '@/components/my-custom-component';

export function useMDXComponents(components: MDXComponents): MDXComponents {
  return {
    // ... existing components
    MyCustomComponent,
    ...components,
  };
}`}
</CodeBlock>

Then use them in your MDX files:

<CodeBlock language="mdx">
{`<MyCustomComponent prop="value">
  Custom content here
</MyCustomComponent>`}
</CodeBlock>
